# Design System

Welcome to the official documentation for our Design System. This system provides a unified set of components, tokens, and guidelines to help you build consistent and accessible user interfaces.

---

## What is this?

This design system is built using React and Material UI, with custom tokens that define our brand's visual identity. It includes:

- **Tokens**: Fundamental building blocks ([Colors](/docs/tokens-colors--docs), [Typography](/docs/tokens-typography--docs), spacing).
- **Components**: Reusable UI elements built with accessibility and performance in mind.
- **Icon Gallery**: A collection of over 50 MUI icons available in our system ([Gallery](/docs/foundations-icon--gallery)).
- **Patterns**: Guidelines on how to combine components to solve common UI problems.

## Getting Started

### For Designers

We use Figma to design all our components and patterns. If you're a designer, check out our Figma library to start designing with our components.

- **[Figma Library](https://figma.com)**
- **[Design Guidelines](https://npxailsa.github.io/design_system)**

### For Developers

To start using the design system in your project, install the package:

```bash
npm install design-system
```

Then, wrap your application with the ThemeProvider to apply the global styles and theme:

```tsx
import { ThemeProvider } from 'design-system';

function App() {
  return (
    <ThemeProvider>
      <YourApp />
    </ThemeProvider>
  );
}
```
