# Typography

Typography is a core element of our design language, providing a clear hierarchy, establishing tone, and creating readable, harmonious interfaces.

## Design Tokens

We use the following typography tokens to render text across our components:

### Typefaces

| Token Name | Description | Value |
| :--- | :--- | :--- |
| `--global-font-primary` | Primary font | F37 Ginger Pro |
| `--global-font-secondary` | Secondary font | Calibri |
| `--brand-font-primary` | Brand primary font alias | `var(--global-font-primary)` |
| `--brand-font-secondary` | Brand secondary font alias | `var(--global-font-secondary)` |

### Font Weights

| Token Name | Description | Value |
| :--- | :--- | :--- |
| `--global-type-weight-thin` | Thin weight | 100 |
| `--global-type-weight-light` | Light weight | 300 |
| `--global-type-weight-default`| Regular weight | 400 |
| `--global-type-weight-medium` | Medium weight | 500 |
| `--global-type-weight-semibold`| Semibold weight | 600 |
| `--global-type-weight-bold` | Bold weight | 700 |
| `--global-type-weight-extra-bold`| Extra bold weight | 800 |

## Principles

Follow these common design principles by using typography in a cohesive, useful, and legible way.

**Design for universal understanding**
Select clear, widely legible typefaces that ensure information is readable across all devices and screen sizes. Use type to establish a consistent hierarchy.

**Balance simplicity and detail to create legibility**
Ensure text contrast meets WCAG 2.1 AA guidelines. Avoid using too many different font sizes and weights within a single view.

**Maintain visual harmony**
Adhere to the typography scale for sizes and line heights. Consistent spacing around text elements creates a harmonious rhythm.

**Use typography intentionally**
Use font weights and sizes to guide the user's eye to the most important information first.

## Using the Component

When using typography in your components, always refer to the design tokens defined in `tokens.css`. Use CSS custom properties rather than hardcoding sizes or typefaces.

```css
.my-title {
  font-family: var(--brand-font-primary);
  font-size: var(--global-type-size-primary-heading-1);
  font-weight: var(--global-type-weight-bold);
  line-height: var(--global-type-line-height-heading-1);
}

.my-text {
  font-family: var(--brand-font-secondary);
  font-size: var(--global-type-size-secondary-body);
  font-weight: var(--global-type-weight-default);
  line-height: var(--global-type-line-height-body);
}
```

## Visual Style

Our typographic style balances the unique character of our brand with functional legibility for application UIs.

### Do
Use standard font weight tokens for consistency.
Use brand aliases (`var(--brand-font-primary)`) rather than global typefaces directly when styling brand-specific components.

### Don't
Do not introduce custom font weights or arbitrary line heights outside the token scale.

## Contribution

If a new typographic scale or size variant is required for a novel use case, consider whether existing sizes can be adapted first. 

To contribute a new typographic variant, please bring it to our Biweekly Design System meeting for review with the whole team.
