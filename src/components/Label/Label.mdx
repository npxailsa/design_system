# Label

The `Label` component is a core building block of the design system, used to display text with optional icons and dropdown split functionality. It is designed to be highly systematic, with 120 possible variants based on size, weight, alignment, and state.

## Usage

```tsx
import { Label } from './Label';
import PersonIcon from '@mui/icons-material/Person';

const MyComponent = () => (
  <Label size="default" weight="bold" leadingIcon={PersonIcon}>
    Example Label
  </Label>
);
```

### Design Tokens

The component uses the following design tokens for styling, which are defined in `src/styles/tokens.css`:

| Token Name | Description | Default Value (Penta) |
| :--- | :--- | :--- |
| `--label-text-color` | Primary text color | `var(--brand-text-main)` |
| `--label-font-family` | Font family | `var(--global-font-primary)` (Archivo) |
| `--label-size-default` | Default font size | `16px` |
| `--label-weight-bold` | Bold font weight | `700` |
| `--label-divider-color` | Dropdown divider color | `var(--brand-border-main)` |

## Guidelines

### Best Practices
- Use the `size` prop to ensure consistent scaling.
- Ensure appropriate contrast ratios when switching between brands.
- Use component tokens for overrides rather than hardcoded values.

### Accessibility
- Ensure labels have sufficient color contrast (WCAG 2.1 AA).
- If an icon carries unique meaning, ensure appropriate ARIA labels are provided via children or parent components.

### Dos and Don'ts
- **Do:** Use systematic weights (bold, medium, light, thin) for consistent hierarchy.
- **Don't:** Overwrite internal component styles directly; use the token system.
