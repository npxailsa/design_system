# Placeholder

The `Placeholder` component is a form-field placeholder UI element — it represents the visual content inside an empty or partially-filled input. It supports 4 sizes, 3 alignments, and 4 types, resulting in 45 possible variants.

## Design Tokens

We use the following tokens to render our placeholders:

### Core Tokens

| Token Name | Description |
| :--- | :--- |
| `--placeholder-text-color` | Primary text and icon color |
| `--placeholder-icon-opacity` | Default opacity for icons |
| `--placeholder-icon-opacity-alt` | Alternative opacity for large size leading icons |
| `--placeholder-counter-opacity` | Opacity for minus-box icon in counter |
| `--placeholder-font-default` | Font family for large, default, and small sizes |
| `--placeholder-font-xs` | Font family for x-small size |
| `--placeholder-counter-color` | Interaction color for counter controls |

## Principles

Follow these common design principles by using placeholders in a cohesive, useful, and legible way.

**Content Clarity**
Placeholders should provide clear hints or examples of what input is expected.

**Legibility**
Ensure placeholder text is legible but clearly distinct from actual user input, typically achieved through reduced opacity or alternative colors.

**Alignment Usage**
Choose alignment that matches the context of the surrounding form or input field.

**Type Selection**
Select the appropriate type (`default`, `italic`, `tags-selected`, or `number`) based on the specific interaction pattern required by the input.

## Using the Component

```tsx
import { Placeholder } from './Placeholder';
import FaceIcon from '@mui/icons-material/Face';

// Default type
const MyComponent = () => (
  <Placeholder size="default" type="default" text="Enter value..." />
);

// Tags selected type
const TagsComponent = () => (
  <Placeholder 
    size="default" 
    type="tags-selected" 
    tags={[{ label: 'Tag 1' }, { label: 'Tag 2', leadingIcon: FaceIcon }]} 
  />
);

// Number type
const CounterComponent = () => (
  <Placeholder 
    size="default" 
    type="number" 
    text="Count"
    value={3}
    onIncrement={() => {}}
    onDecrement={() => {}}
  />
);
```

### Do

Use systematic sizes and types for consistent form field patterns.

### Don't

Overwrite internal component styles directly; use the token system. Avoid creating invalid combinations (like x-small tags-selected).

## Contribution

When proposing updates or new properties for the Placeholder component, consider the following:
1. Does the proposed change align with existing form field patterns?
2. Does it maintain backward compatibility with our systematic sizing approach?

To contribute a new variation, please bring it to our Biweekly Design System meeting for review with the whole team.
